# Financial Health Risk — 財務健康風險分析

系統性地從流動性、獲利能力、槓桿/償債、現金流、Altman Z-Score、成長性 6 個維度分析財務健康狀況，產出風險評估報告與改善建議。

## 此技能的功能

- 引導使用者提供財務文件或數字
- 自動判斷是否需要呼叫 `scripts/extract_financials.py` 解析 PDF
- 依序執行 7 個分析階段，每階段產出結構化結果
- 對照 `references/industry-benchmarks.md` 進行行業比較
- 依 `references/risk-scoring-rules.md` 計算各維度評分
- 整合輸出完整的 Markdown 風險報告與改善建議

## 適用時機

- 企業主評估自家公司財務健康狀況
- 投資人分析上市/非上市公司財報
- 銀行/信貸機構評估借款方財務風險
- 財務顧問為客戶準備財務診斷報告

## 觸發範例

以下輸入會啟動此技能：
- 「幫我分析這份財報 PDF」
- 「這家公司財務健不健康？」
- 「做一個財務健康檢查」
- 「financial risk analysis for this company」
- 「看一下這些財務數字有沒有問題」
- 「計算一下 Altman Z-Score」

啟動後，先收集以下資訊：

```
請提供以下資訊：

1. 財務文件（必填）：
   - 本地 PDF 路徑（如 /path/to/report.pdf）
   - PDF 網址（如 https://example.com/annual_report.pdf）
   - 財務網頁網址（如 Yahoo Finance、公開資訊觀測站）
   - 直接貼上財務數字（損益表、資產負債表、現金流量表）
2. 公司名稱（選填）：用於報告標題
3. 產業別（選填）：用於行業基準比較
   可選：科技製造、傳統製造、零售/流通、服務業、建設/不動產、金融服務、其他
4. 分析期間（選填）：例如 2024 年度、2024Q3
5. 是否有多年數據（選填）：提供前一年數據可進行成長性趨勢分析
```

## 執行流程

### Phase 1：資料抽取與驗證

**目標**：取得結構化財務數據，確認數據完整性

執行步驟：

1. **判斷輸入類型**：
   - 若為本地 PDF 路徑 → 執行 `python scripts/extract_financials.py <pdf_path>`
   - 若為 PDF URL（http/https 且 Content-Type 為 pdf）→ 執行 `python scripts/extract_financials.py <url>`（腳本自動下載）
   - 若為財務網頁 URL → 執行 `python scripts/extract_financials.py <url>`（腳本解析 HTML 表格）
   - 若為直接貼上的數字 → 整理為標準格式
   - 若 AI 可直接讀取 PDF → 優先直接讀取，腳本作為 fallback

2. **驗證數據完整性**：
   - 損益表：營業收入、淨利（必要）；毛利、營業利益、利息費用（選填）
   - 資產負債表：流動資產、流動負債、總資產、總負債、股東權益（必要）；現金、存貨、保留盈餘（選填）
   - 現金流量表：營業活動現金流、資本支出（選填，缺少時降低信心度）

3. **設定報告信心度**：
   - 三張報表齊全 → 高信心度
   - 缺少現金流量表 → 中信心度
   - 僅有一張報表 → 低信心度

4. **確認分析版本**：
   - 有市值數據 → 使用 Altman Z-Score 上市版
   - 無市值數據 → 使用 Z'-Score 非上市版
   - 服務業 → 使用 Z''-Score

### Phase 2：流動性分析

**目標**：評估短期償債能力

參照 `references/ratios-formulas.md` 計算：
- 流動比率 = 流動資產 ÷ 流動負債
- 速動比率 = (流動資產 − 存貨) ÷ 流動負債
- 現金比率 = 現金及約當現金 ÷ 流動負債

對照 `references/industry-benchmarks.md` 選取行業基準比較。
依 `references/risk-scoring-rules.md` 計算流動性維度分數（滿分 100）。

輸出：各指標數值、行業比較、燈號、維度分數

### Phase 3：獲利能力分析

**目標**：評估企業賺錢能力與效率

計算：
- ROE = 稅後淨利 ÷ 平均股東權益 × 100%
- ROA = 稅後淨利 ÷ 平均總資產 × 100%
- 毛利率 = (營收 − 營業成本) ÷ 營收 × 100%
- 營業利益率 = 營業利益 ÷ 營收 × 100%
- 淨利率 = 稅後淨利 ÷ 營收 × 100%

**備註**：平均值需要兩期數據；若僅有單期，使用期末值並標注。

輸出：各指標數值、行業比較、燈號、維度分數

### Phase 4：槓桿／償債分析

**目標**：評估長期財務結構與償債能力

計算：
- 負債比率 = 總負債 ÷ 總資產 × 100%
- D/E Ratio = 總負債 ÷ 股東權益
- 利息保障倍數 = 營業利益（EBIT）÷ 利息費用（無利息費用則標記 N/A）

輸出：各指標數值、行業比較、燈號、維度分數

### Phase 5：現金流分析

**目標**：評估盈利品質與實際現金產生能力

計算：
- CFO/Sales = 營業活動現金流 ÷ 營收 × 100%
- FCF = 營業活動現金流 − 資本支出
- FCF/Net Income = FCF ÷ 稅後淨利

**若無現金流量表**：跳過此 Phase，在報告中標注「現金流數據不足，信心度降低」。

輸出：各指標數值、燈號、維度分數

### Phase 6：Altman Z-Score ＋ 成長性

**目標**：計算破產預測分數，並分析成長趨勢

**Altman Z-Score**（依 Phase 1 確認的版本計算）：

上市公司版：
```
Z = 1.2×X1 + 1.4×X2 + 3.3×X3 + 0.6×X4 + 1.0×X5
X1 = (流動資產 − 流動負債) / 總資產
X2 = 保留盈餘 / 總資產
X3 = EBIT / 總資產
X4 = 股票市值 / 總負債帳面值
X5 = 營收 / 總資產
```

非上市公司版：
```
Z' = 0.717×X1 + 0.847×X2 + 3.107×X3 + 0.420×X4 + 0.998×X5
（X4 改用股東權益帳面值 / 總負債帳面值）
```

**成長性分析**（需兩期數據）：
- 營收 YoY = (本期營收 − 前期營收) ÷ 前期營收 × 100%
- 淨利 YoY = (本期淨利 − 前期淨利) ÷ |前期淨利| × 100%
- 資產 YoY = (本期總資產 − 前期總資產) ÷ 前期總資產 × 100%

若無前期數據，成長性維度標記「數據不足」，權重重新分配至其他維度。

### Phase 7：綜合風險報告整合

**目標**：整合所有分析，產出完整報告

1. 依 `references/risk-scoring-rules.md` 計算各維度加權分數
2. 計算綜合風險分數
3. 判定風險等級（低風險 / 中風險 / 高風險 / 危急）
4. 檢查特殊旗標（流動性危機、盈利品質差、高槓桿警示等）
5. 產出改善建議（短/中/長期）
6. 輸出完整 Markdown 報告

## 錯誤處理

| 情況 | 處理方式 |
|------|---------|
| PDF 無法解析 | 請使用者直接貼上財務數字，或確認 PDF 非掃描版 |
| 關鍵數據缺失（如無淨利） | 標注缺失，跳過相關指標，降低信心度 |
| 金融業財報 | 說明一般指標不適用，改用銀行業專用指標（NIM、NPL、CAR） |
| 僅有單期數據 | 跳過成長性分析，其他維度正常執行 |
| 數字單位不明 | 詢問使用者確認（元、千元、百萬元、億元） |
