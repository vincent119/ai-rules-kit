# Data Visualization Guide — 資料視覺化圖表選擇與資訊設計指南

info-architect 在設計資料視覺化時使用的圖表選擇矩陣、資訊層級原則與視覺化最佳實踐。

## 此技能的功能

- 提供依資料類型選擇圖表的決策矩陣
- 提供每種圖表的設計規範
- 提供 LATCH 資訊組織原則
- 提供數字格式化與色彩無障礙標準

## 適用時機

- 決定如何呈現數據時
- 設計資訊架構與視覺層級
- 搭配 `pres-presentation-designer` 技能使用

## 觸發範例

- 「我有 5 年的銷售趨勢數據，用什麼圖表？」
- 「要比較 4 個產品的 3 個指標，怎麼呈現？」
- 「這頁有很多數字，怎麼讓它看起來清楚？」

## 圖表選擇矩陣

### 依資料關係選擇

| 資料關係 | 推薦圖表 | 資料點數 | 避免 |
|---------|---------|---------|------|
| **比較** | 長條圖、橫條圖 | 3~12 項 | 圓餅圖 |
| **趨勢** | 折線圖、面積圖 | 5+ 個時間點 | 長條圖（時間點少時除外） |
| **組成** | 圓餅圖、堆疊長條圖 | 3~6 個類別 | 超過 7 類改用橫條圖 |
| **分布** | 直方圖、箱型圖 | 連續數據 | 折線圖 |
| **關聯** | 散點圖、泡泡圖 | 兩變數關係 | 折線圖 |
| **流程** | 桑基圖、漏斗圖 | 階段轉換 | 圓餅圖 |
| **地理** | 地圖、熱力圖 | 地區數據 | 長條圖 |
| **排名** | 橫條圖、斜率圖 | 順序比較 | 圓餅圖 |

### 依資料點數選擇

| 資料點數 | 推薦圖表 |
|---------|---------|
| 1 個數字 | 大數字（Big Number）+ 說明文字 |
| 2~3 個 | 比較長條圖、並排數字 |
| 4~12 個 | 長條圖、橫條圖、折線圖（6 個以下用圓餅） |
| 13~50 個 | 折線圖、散點圖 |
| 50+ 個 | 熱力圖、散點圖、直方圖 |

## 各圖表設計規範

### 長條圖 / 橫條圖

- Y 軸從 0 開始（截斷 Y 軸會誤導讀者）
- 長條寬度：間距的 1.5~2 倍
- 排序：依大小排序（時間序列除外）
- 色彩：1 個強調色 + 中性色
- 超過 12 個類別改用橫條圖

### 折線圖

- 線條粗細：2~3px（主線）、3~4px（強調線）
- 資料點：10 個以下顯示點，超過隱藏
- Y 軸：從合理最小值開始（非必須從 0）
- 多條線：最多 4~5 條（超過需拆圖）

### 圓餅圖 / 環形圖

- 類別數：3~6 個（超過 7 個改用橫條圖）
- 起始位置：12 點鐘方向
- 最大類別：放在 12 點鐘位置
- 標籤：直接標在扇形上，避免圖例
- 禁用 3D 圓餅圖

### 散點圖

- 資料點大小：一致（泡泡圖除外）
- 趨勢線：加上 R² 值說明相關強度
- 離群值：明確標注並說明

## LATCH 資訊組織原則

依據資訊的本質選擇最適合的組織方式：

| 原則 | 說明 | 適用情境 |
|------|------|---------|
| **L** Location（位置） | 依地理或空間組織 | 地區比較、空間分布 |
| **A** Alphabet（字母） | 依字母順序組織 | 術語表、名稱列表 |
| **T** Time（時間） | 依時間順序組織 | 歷史沿革、專案時程、趨勢 |
| **C** Category（類別） | 依類型或主題組織 | 功能分類、產品分類 |
| **H** Hierarchy（層級） | 依重要性或規模組織 | 排名、組織架構、優先級 |

## 資料說故事結構

### 三段式結構

1. **背景**：「目前的狀況是...」（提供脈絡）
2. **洞察**：「關鍵發現是...」（核心 Insight）
3. **建議**：「因此我們應該...」（行動方向）

### 投影片敘事模式

| 模式 | 投影片流程 |
|------|-----------|
| **總分** | 總覽 → 各項細節 |
| **歸納** | 各項細節 → 總結規律 |
| **比較** | 選項 A → 選項 B → 結論 |
| **時間** | 過去 → 現在 → 未來 |
| **問題-解答** | 問題 → 分析 → 解方 |

## 數字格式化規範

| 數字類型 | 格式 | 範例 |
|---------|------|------|
| 金額（大） | 單位縮寫 | $1.2M、$350K |
| 金額（小） | 完整數字 + 千分位 | $12,500 |
| 百分比 | 1 位小數 | 23.4% |
| 成長率 | +/- 符號 + 色彩 | +12.3%（綠）、-5.1%（紅） |
| 日期 | YYYY-MM 或 MM/YYYY | 2024-Q3 |
| 大數字 | K/M/B 縮寫 | 1.2M 用戶 |

## 色彩無障礙標準

### 圖表色彩選擇

| 類型 | 用途 | 原則 |
|------|------|------|
| 順序色（Sequential） | 數值大小 | 淺色（低）→ 深色（高） |
| 發散色（Diverging） | 正負對比 | 負（冷色）← 中性 → 正（暖色） |
| 類別色（Categorical） | 不同類別 | 最多 6 種，色相差異明顯 |

### 無障礙檢查

| 項目 | 標準 |
|------|------|
| 色盲友善 | 避免單純紅綠對比，搭配形狀或紋理 |
| 對比度 | 文字與背景最低 4.5:1（WCAG AA） |
| 色盲模擬 | 使用工具驗證（Coblis、Color Oracle） |
| 圖案輔助 | 黑白列印時仍可區分不同類別 |

## 投影片資訊密度建議

| 投影片類型 | 文字量 | 圖表數 | 原則 |
|-----------|--------|--------|------|
| 標題頁 | 極少 | 0 | 視覺衝擊優先 |
| 觀點頁 | 少（1 句話） | 0~1 | 一頁一訊息 |
| 數據頁 | 中（3~5 點） | 1 | 圖表為主，文字輔助 |
| 說明頁 | 中 | 0~1 | 條列清晰 |
| 附錄頁 | 多 | 1~2 | 密度可提高 |

**黃金法則**：每頁只傳達一個核心訊息，其他都是支撐。
